# ERC-8004: Agent Identity Standard

## What is ERC-8004?

ERC-8004 is an Ethereum standard for AI agent identity. It provides:

- **Soulbound NFT identity**: Each agent gets a non-transferable token tied to its owner wallet
- **On-chain metadata**: Agent name, description, capabilities, and service endpoints stored via tokenURI
- **Reputation registry**: Separate contract for on-chain feedback and trust scoring
- **Multi-chain**: Same contract addresses deployed across 20+ EVM chains via EIP-1967 proxies

## Contracts

| Contract | Address | Purpose |
|----------|---------|---------|
| IdentityRegistry | `0x8004A169FB4a3325136EB29fA0ceB6D2e539a432` | Agent registration + metadata |
| ReputationRegistry | `0x8004BAa17C55a88189AE136b182e5fdA19dE9b63` | Feedback + trust scores |

These are deployed at the same vanity addresses across Base, Arbitrum, Ethereum, Optimism, Polygon, and 15+ other chains.

## Events Indexed

### IdentityRegistry

| Event | Description |
|-------|-------------|
| `Transfer(from, to, tokenId)` | Mint (from=0x0) or transfer of agent identity |
| `Registered(agentId, owner, agentURI)` | New agent registration with metadata URI |
| `MetadataSet(agentId, metadataKey, metadataValue)` | Key-value metadata update |

### ReputationRegistry

| Event | Description |
|-------|-------------|
| `NewFeedback(agentId, client, score, comment)` | Reputation feedback submitted |

## Metadata Format

The `tokenURI` returns a JSON object following the ERC-8004 registration spec:

```json
{
  "type": "https://eips.ethereum.org/EIPS/eip-8004#registration-v1",
  "name": "My Agent",
  "description": "What my agent does",
  "image": "https://example.com/logo.png",
  "services": [
    { "name": "web", "endpoint": "https://myagent.com" },
    { "name": "OASF", "endpoint": "https://github.com/...", "version": "0.8.0" }
  ],
  "x402Support": true,
  "active": true,
  "supportedTrust": ["reputation"],
  "registrations": [
    {
      "agentId": 100,
      "agentRegistry": "eip155:8453:0x8004A169FB4a3325136EB29fA0ceB6D2e539a432"
    }
  ]
}
```

## How AgentZone Uses ERC-8004

1. **Indexing**: Envio HyperIndex watches all events on both contracts
2. **Metadata Resolution**: API calls `tokenURI()` via RPC and resolves the returned URI
3. **Trust Scoring**: Aggregates reputation feedback into trust scores
4. **Verification**: Frontend shows verification badges for agents with on-chain identity
5. **Discovery**: JSON-LD API exposes agent data in machine-readable format
